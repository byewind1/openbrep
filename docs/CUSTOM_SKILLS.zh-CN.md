# 自定义 Skill 使用说明

自定义 Skill 用来保存你的项目经验，让 AI 在后续生成或修改 GDL 对象时自动遵守这些规则。

它适合记录：

- 项目或构件类型，例如门窗、斗拱、家具、栏杆；
- 参数命名规范；
- 常用 GDL 结构和代码模式；
- 材质、笔号、线型等项目约定；
- 容易出错的规则或检查清单。

## 在 chat 里创建 Skill

直接在 AI 助手聊天区输入类似句子：

```text
帮我创建一个门窗项目 skill
```

也可以这样说：

```text
让 AI 学习我的斗拱构件规范
```

```text
我想建立一个书架类 GDL 生成规则
```

AI 会进入多轮引导，逐步询问项目类型、参数规则、常用模式等信息。

## 创建时建议提供什么

信息越明确，Skill 越容易自动命中并发挥作用。

建议至少提供：

1. **项目或构件类型**  
   例如：铝合金门窗、古建斗拱、模块化书架。

2. **触发关键词**  
   例如：门窗、窗户、铝合金窗框、双扇窗。

3. **参数命名规则**  
   例如：宽度用 `A`，深度用 `B`，高度用 `ZZYZX`，窗框厚度用 `frame_thk`。

4. **常用结构模式**  
   例如：窗框用四边 `BLOCK` 组合，重复层板用 `FOR/NEXT`。

5. **材质与属性约定**  
   例如：主框材质用 `mat_frame`，玻璃材质用 `mat_glass`。

6. **禁止事项或踩坑**  
   例如：不要硬编码尺寸；新增参数必须同步写入 `paramlist.xml`。

## 生成 Skill 文件

当信息补充得差不多时，直接输入：

```text
生成
```

AI 会把这轮对话整理成一个 Skill 文件，并保存到项目的 `skills/` 目录。

## 创建后如何生效

创建完成后，后续正常提需求即可，不需要再输入 Skill 文件名。

例如刚创建了门窗 Skill，后续可以直接说：

```text
生成一个铝合金窗户
```

或：

```text
把这个窗户改成双扇结构
```

系统会根据 Skill 里的“触发关键词”和“适用场景”自动加载相关 Skill，并注入到生成/修改流程。

## 注意事项

- 一个 Skill 尽量只覆盖一个项目或一个构件体系，不要把无关规则混在一起。
- 触发关键词要具体，例如“门窗、窗户、铝合金窗框”比“建筑构件”更有效。
- 不要把 API Key、客户隐私、商业合同、未公开项目资料写进 Skill。
- 如果后续生成结果没有明显体现 Skill，可继续在 chat 中补充规则，再重新生成一个更明确的 Skill。

## 示例对话

```text
用户：帮我创建一个门窗项目 skill
AI：请告诉我这个门窗项目主要有哪些构件和命名规则？
用户：主要是铝合金窗，宽度用 A，高度用 ZZYZX，窗框厚度用 frame_thk，玻璃材质用 mat_glass。
AI：常用结构或禁止事项有哪些？
用户：窗框用四边 BLOCK 组合，不要硬编码尺寸，新增参数必须同步 paramlist.xml。
用户：生成
```

之后可以直接输入：

```text
生成一个铝合金双扇窗
```

AI 会自动尝试加载刚创建的门窗 Skill。
