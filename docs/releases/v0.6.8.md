# OpenBrep v0.6.8

Release type: macOS installer startup fix.

## Why

The v0.6.7 macOS package exposed a packaging bug in the desktop launcher. The
launcher used `sys.executable -m streamlit` after PyInstaller had frozen the
app. In a frozen build, `sys.executable` points back to the packaged OpenBrep
executable, not to a normal Python interpreter. That could recursively launch
OpenBrep instead of starting Streamlit cleanly.

## What changed

- Fix the packaged launcher:
  - Run Streamlit in-process through `streamlit.web.cli`.
  - Avoid spawning `sys.executable -m streamlit` from a frozen executable.
  - Bind to `127.0.0.1` on an available local port.
  - Open the browser after the Streamlit health endpoint becomes available.
- Improve the macOS zip layout:
  - Add `OpenBrep.command` for a clearer double-click entry point.
  - Add `README-macOS.txt` inside the package.
  - Document the macOS right-click Open path for unsigned builds.
- Make the UI install extra complete:
  - Add `cryptography` to `openbrep[ui]` so UI imports do not depend on hidden
    local environment state.
- Add packaging launcher regression tests.
