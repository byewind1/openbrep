# OpenBrep v0.6.6

Release type: UI usability patch release.

## Why

This release packages a focused round of UI cleanup after the v0.6.5 installer
automation patch. The goal is to reduce cognitive load in the Streamlit workbench
and make the common OpenBrep loop more direct:

```text
open source -> preview/edit -> compile GSM -> continue with AI
```

## What changed

- Split the old unified open control into direct `Open file` and `Open HSF
  project` actions.
- Remove the extra file/folder routing dialog and use native file or folder
  pickers directly.
- Activate macOS native file/folder dialogs before showing them, reducing the
  need to click the chooser window before selecting.
- Move the preview workbench above script editing so generated geometry is
  visible earlier in the workflow.
- Put script editing and object parameters behind adjacent tabs, reducing
  vertical scrolling.
- Simplify GSM compilation to one `Compile GSM` action:
  selected output folder when chosen, default `workspace/output` when cancelled.
- Hide successful compile detail output; keep success as a toast and show errors
  only when compilation fails.
- Move long import/load logs out of the LLM chat area and remove low-value
  controls such as pending-review prompts, chat feedback buttons, preview
  settings, manual script check, reset project, and compile log buttons.
- Keep the error-learning workflow visible through the `整理错题本` action.
- Move Archicad/Tapir linkage to the bottom of the workspace panel for now,
  pending a later real-device review.
