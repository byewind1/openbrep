# OpenBrep v0.6.7

Release type: UI cleanup closing patch.

## Why

This release closes the current UI cleanup round. After v0.6.6 simplified the
main open, preview, edit, and compile flow, v0.6.7 removes the remaining
low-frequency controls that were still competing with the everyday OpenBrep
workflow.

The intended primary loop is now:

```text
open or generate HSF -> preview -> edit scripts/parameters -> save HSF -> compile GSM
```

## What changed

- Add an explicit HSF save model:
  - `保存 HSF` silently saves back to the current HSF source directory.
  - Unsaved projects route directly into `另存为 HSF`.
  - `另存为 HSF` lets the user choose a parent directory and HSF folder name.
- Hide revision management from the main UI:
  - Remove the visible `版本管理`, `刷新历史`, `保存版本`, and `恢复此版本`
    controls.
  - Stop automatic hidden revision snapshots after successful compile.
  - Keep the underlying revision code available for future advanced history
    work.
- Hide the Archicad/Tapir live-link UI:
  - Remove the left-panel `Archicad 实机联动` controls.
  - Hide the parameter-panel `Archicad 写回` tab in the default UI.
  - Remove the sidebar Archicad-not-running warning.
  - Keep the Tapir bridge/controller code in place for later review.
- Improve chat record UX:
  - Chat records open in a dialog instead of filling the LLM panel.
  - Record detail now returns to the record list instead of closing the browser.
  - Delete requires confirmation.
  - Preview and restore actions close stale chat-record dialogs before running.
