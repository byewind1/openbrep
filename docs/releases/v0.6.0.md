# OpenBrep v0.6.0

发布日期：2026-04-06

## 版本定位

v0.6.0 不是“全部平台化完成版”，而是 Runtime 独立 Phase 1 主骨架的收尾发布版。

它的重点不是继续扩功能，而是把 create / modify / repair / chat 主链路统一到 runtime，并把当前版本推进到可受控上线的状态。

## 核心变化

- `TaskPipeline` 成为 create / modify / repair / chat 的统一执行中心
- repair 独立 intent 链路闭合，不再借 `DEBUG` 语义复用 modify 路径
- CLI 与 UI 的行为一致性继续提升
- trace、回归测试、UI 写回体验继续补齐
- generation 结果解释逻辑进一步从 `ui/app.py` 往 runtime 收口

## 发布判断

- 可以受控上线
- 适合继续真实场景验证 / 内测 / 小范围生产验证
- 不代表 v0.7 工具化已经开始
- 不建议对外宣称 Runtime 独立 Phase 1 已彻底收官

## 已知未完成项

- `ui/app.py` 仍需继续瘦身
- `core.py` 仍有进一步拆边界空间
- v0.6 最小回归基线还需继续制度化
