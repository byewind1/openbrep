# OpenBrep v0.6.0

发布日期：2026-04-06

## 版本定位

v0.6.0 是 OpenBrep Runtime Phase 1 主骨架的正式发布收尾版。

它的重点不是继续扩功能，而是把 create / modify / repair / chat 主链路统一到 runtime，并把代码版本、UI 展示、README、安装说明与发布说明全部收平，使 `v0.6.0` 作为正式版本成立。

## 核心变化

- `TaskPipeline` 成为 create / modify / repair / chat 的统一执行中心
- repair 独立 intent 链路闭合，不再借 `DEBUG` 语义复用 modify 路径
- CLI、UI 与 runtime 的主行为和版本口径完成统一
- trace、回归测试、UI 写回体验补齐到正式发布所需基线
- generation 结果解释逻辑进一步从 `ui/app.py` 往 runtime 收口

## 发布判断

- `v0.6.0` 可作为正式版本发布
- Runtime Phase 1 主骨架已经完成收口
- 后续仍可继续优化 UI 与 core 边界，但这不影响 `v0.6.0` 的发布成立
- v0.7 的工具化工作将在后续版本中继续推进

## 后续方向

- `ui/app.py` 继续瘦身
- `core.py` 继续拆边界
- v0.6 最小回归基线继续制度化
