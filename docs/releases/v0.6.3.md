# OpenBrep v0.6.3

发布日期：2026-04-28

## 版本定位

v0.6.3 是基于 `v0.6.2` 的工作台治理与自我提升版本。

本次发布重点不是扩展单点生成能力，而是把 OpenBrep 从“一次性 GDL 生成工具”推进为更可长期使用的 GDL 工作台：项目目录更稳定，版本记录更贴近具体 HSF/GSM 项目，真实 Archicad 错误可以沉淀为个人工作空间记忆，并在后续生成中作为约束注入。

## 核心变化

- 新增个人工作空间记忆层：
  - 聊天记录持久化到 `<work_dir>/.openbrep/memory/chats/chat_transcript.jsonl`
  - 用户真实错误与编译错误沉淀到 `<work_dir>/.openbrep/memory/learnings/error_lessons.jsonl`
  - 用户主动整理后生成 `<work_dir>/.openbrep/memory/skills/learned_skill.md`
- 新增 GDL 错题本自我提升链路：
  - 支持从用户聊天中的脚本错误片段自动识别错误线索
  - 支持用户点击“整理错题本”后扫描聊天记录并压缩为自我提示
  - 提示词注入去除无复用价值的具体行号，只保留错误模式、脚本范围与约束
- LLM 注入分层：
  - 用户工作空间记忆优先
  - 工作空间近期错题其次
  - 源码内开发者基线作为兜底
- HSF / 项目管理治理：
  - 编译未命名项目时持久化稳定 HSF 目录
  - 版本记录按当前项目作用域管理
  - 版本记录包含编译名，便于识别具体 GSM 项目版本
- UI 与操作逻辑治理：
  - Streamlit 主界面继续拆分，降低 `ui/app.py` 维护压力
  - 聊天渲染与聊天控制链路进一步收敛
  - 新增“整理错题本”工作区入口
- 配置与生成细节修复：
  - 修复自定义 provider 切换时 `base_url` 等配置同步问题
  - 改进余额不足等 LLM 错误提示
  - 参数表展示去掉容易误导的单位标注，避免名称单位与 Archicad 参数单位不一致

  - `AGENTS.md`

## 发布判断

`v0.6.3` 可以作为小版本发布。它的核心价值是让 OpenBrep 的项目管理、版本治理、聊天记忆和 GDL 错题本形成可持续闭环，为后续发展为顶级 GDL 代码工作台打基础。
