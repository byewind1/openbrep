# OpenBrep v0.6.9

Release type: package verification patch.

## Why

Developers often already have `obr` installed locally, so testing with `obr`
does not prove that a GitHub Release zip works. This patch adds a package-level
smoke path that runs the executable inside the downloaded zip directly.

## What changed

- Add package smoke testing support to the launcher:
  - `OPENBREP_PORT` / `STREAMLIT_SERVER_PORT` can force a known local port.
  - `OPENBREP_NO_BROWSER=1` disables automatic browser opening for automation.
- Add `scripts/package_smoke.py`:
  - Extracts a Release zip into a temporary directory.
  - Runs `OpenBrep/OpenBrep.command`, `OpenBrep/OpenBrep`, or `OpenBrep.exe`
    from inside the package.
  - Checks `http://127.0.0.1:<port>/_stcore/health`.
  - Terminates the package process and cleans the temporary directory.
- Add regression tests for the launcher smoke-test controls.

## Verification

- `python -m pytest tests/ -q`
- `python -m pip wheel . -w /tmp/openbrep_wheel_check --no-deps`
- `ruby -e 'require "yaml"; YAML.load_file(".github/workflows/build-installers.yml"); puts "yaml ok"'`
- After publishing:
  - `gh release download v0.6.9 --pattern OpenBrep-free-macOS.zip --dir /tmp/openbrep-v069-check --clobber`
  - `python scripts/package_smoke.py /tmp/openbrep-v069-check/OpenBrep-free-macOS.zip`
