# OpenBrep v0.6.11

Release type: macOS installer packaging fix.

## Compatibility

- macOS package: Apple Silicon only (`arm64`, M1/M2/M3/M4).
- Minimum macOS: macOS 14 Sonoma or later.
- Intel Mac is not covered by the current `OpenBrep-free-macOS.zip` asset.
- This was checked against the published GitHub Release zip by scanning the
  frozen package binaries, not just the local development environment.

## Why

The v0.6.10 Release zip could pass a simple health check while still failing
in real browser use. The frozen package was missing Streamlit runtime pieces
that the UI needs at startup and on first page load.

## What changed

- Packaged launcher:
  - Keeps the fixed-port and browser-disable controls.
  - Still launches the frozen Streamlit app in-process.
- Frozen package contents:
  - Bundles `streamlit/static` at the runtime path Streamlit expects.
  - Bundles `streamlit_ace/frontend/build` at the runtime path the component expects.
  - Includes `streamlit.runtime.scriptrunner` hidden imports so `magic_funcs` resolves.
- Verification tooling:
  - `scripts/package_smoke.py` checks health plus homepage.
  - `scripts/package_browser_smoke.py` checks the real browser load path and logs.
- Release docs and version metadata updated to `0.6.11`.

## Verification

- `python -m pytest tests/test_packaging_launcher.py tests/test_package_smoke.py -q`
- `python -m pytest tests/ -q`
- `python -m pip wheel . -w /tmp/openbrep_wheel_check --no-deps`
