# OpenBrep v0.7.0

Release date: 2026-05-23

Release type: GDL asset lifecycle platform milestone.

## Compatibility

- Archicad: AC28 / AC29 through LP_XMLConverter when available.
- GDL Copilot Add-On: AC29 macOS.
- Python: 3.10+.
- macOS package: Apple Silicon only (`arm64`, M1/M2/M3/M4), macOS 14 Sonoma or later.
- Intel Mac is not covered by the current `OpenBrep-free-macOS.zip` asset.
- The current macOS zip is not yet Developer ID signed and notarized. If
  Gatekeeper blocks the app, unzip it and run:
  `xattr -dr com.apple.quarantine /path/to/OpenBrep`, then start
  `OpenBrep.command` again. A signed and notarized package is planned.

## Why

v0.7.0 moves OpenBrep from a GDL generation tool toward a traceable GDL asset
development workbench. The core product contract now covers versioned HSF
source snapshots, engineering-level change explanations, and before/after
compile comparison.

## What changed

- Version management:
  - `modify` / `repair` now create a before revision before mutation.
  - Successful compile creates an after revision with full HSF snapshot metadata.
  - Added top-level CLI commands:
    - `obr history <project>` lists revision history.
    - `obr rollback <project> <revision>` restores a selected revision while
      saving the current state first.
  - Revision manifests include v0.7 lifecycle fields: `trigger`, `intent`,
    `user_instruction`, `changed_files`, `parent_revision_id`, and `compile`.
- Change explanation:
  - `modify` output now includes an engineering summary of changed scripts,
    unchanged scripts, parameter additions/removals/updates, linter auto-fix
    count, and compile result.
  - `GDLContractChecker` is integrated into modify plain-text output.
  - Contract errors are marked with `ERROR`, warnings with `WARNING`, and the
    after revision metadata records contract issue counts.
- Compile comparison:
  - CLI `modify` supports `--compare mock` and `--compare real`.
  - Comparison output reports compile status changes, `.gsm` size changes, and
    parameter count changes.
  - Comparison data is written into trace and after revision metadata.
- Harness and scorecards:
  - Benchmark tasks now consume YAML success criteria for required parameters,
    required scripts, and basic GDL assertions.
  - Scorecards support mock / real / auto compiler modes.
  - Added GDL contract checks, semantic assertion DSL, JSONL trend reporting,
    and markdown benchmark summaries.
- Knowledge and skills:
  - Extracted the modify skills prompt into `prompts/tasks/modify_skills.md`.
  - Added the dougong 3D GDL script and skill as a real object case.
