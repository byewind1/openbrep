# OpenBrep v0.6.5

Release type: installation release automation patch.

## Why

The v0.6.4 installer workflow successfully built the macOS / Windows artifacts,
but the release publishing step failed because GitHub CLI does not support using
`gh release create --notes-from-tag` together with `--repo`.

Published release tags are immutable in the OpenBrep release process, so this
patch keeps `v0.6.4` as-is and publishes a follow-up patch tag.

## What changed

- Replace `gh release create --notes-from-tag` with
  `gh release create --generate-notes`.
- Keep explicit `--repo "$GH_REPO"` so the release job works without a checkout
  step.
- Bump package version to `0.6.5`.
- Keep the v0.6.4 installation UX changes intact:
  GitHub Release desktop packages first, `pipx` / `uv tool` second, source
  install for developers.

## Verification

- YAML parses successfully.
- `python -m pytest tests/ -q`
- `python -m pip wheel . -w /tmp/openbrep_wheel_check --no-deps`
