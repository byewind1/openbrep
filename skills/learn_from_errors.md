# Skill: learn_from_errors（GDL 错题本）

> 目标：把真实 Archicad 编译/运行错误和用户纠错转化为可复用经验，减少同类错误重复发生。

## When to Use / 适用场景

当任务包含以下任一情况时启用：

- 用户粘贴 Archicad / LP_XMLConverter / Tapir 的真实错误。
- 用户用自然语言总结“刚才哪里错了”。
- 自动编译失败，需要修复并记录原因。
- 同类错误在多个项目或多次生成中反复出现。

## Strategy / 策略

1. **先分类**：控制流闭合、命令参数、变量映射、数值保护、参数 XML、变换平衡、2D 表达、通用编译错误。
2. **再定位**：优先找文件、行号、相关参数、最近的控制块或变换块。
3. **最小修复**：只改导致错误的最小脚本范围，不重写无关逻辑。
4. **沉淀经验**：把错误模式变成“下次生成前必须规避”的约束。
5. **回归验证**：修复后至少检查参数引用、结构闭合、ADD/DEL 配平、命令参数数量。

## Rules / 规则

- 真实错误优先级高于通用知识库和模型记忆。
- 用户纠错视为高价值信号，不要辩解；先吸收，再修正。
- 错题约束必须是可执行的生成约束，不写空泛总结。
- 同类错误重复出现时，提高约束强度，生成前先做对应自检。
- 不要把一次偶然错误无限扩大成全局禁止规则；重复出现或用户确认后再强化。

## Common Patterns / 高频模式

### 变量映射错误

错误信号：

```text
Undefined variable width
使用了未初始化变量
```

约束：

- `width/depth/height` 不是默认 GDL 参数名。
- 宽/深/高应使用 `A/B/ZZYZX`。
- 派生变量必须先在 `1d.gdl` 中赋值，再在 2D/3D 中使用。

### 命令参数错误

错误信号：

```text
Wrong number of arguments
```

约束：

- 写 `PRISM_`、`REVOLVE`、`TUBE` 前先核对命令签名。
- `PRISM_` 必须包含高度参数。
- mask、点数量、坐标数量必须一致。

### 控制流闭合错误

错误信号：

```text
ENDIF expected
NEXT expected
Unexpected ENDIF
```

约束：

- 多行 `IF THEN` 必须有 `ENDIF`。
- 单行 `IF cond THEN var = value` 不要额外加 `ENDIF`。
- `FOR` 必须有 `NEXT`。

## Output Discipline / 输出纪律

修复回复必须包含：

1. 错误分类
2. 根因
3. 最小修复点
4. 防止复发的生成约束

不要只说“已修复”。
